# Kumix UI

React component packages for Kumix products. The publishable packages in this repository are `@kumix/ui` (Radix-based) and `@kumix/shadcn` (Base UI + shadcn registry).

The package is built with Bun, TypeScript, tsdown, Radix UI primitives, Tailwind utility classes, and related headless UI libraries.

## Package Entries

- `@kumix/ui` - all exported React components.
- `@kumix/ui/hooks` - hooks-only entry.
- `@kumix/ui/css` - CSS entry emitted as `dist/style.css`.

The root component entry and hooks entry are client-side React entries. Components are written for React applications that support client components.

## Installation

```bash
bun add @kumix/ui
```

Install the peer dependencies used by the components you consume. The package declares these peers:

```bash
bun add @kumix/utils class-variance-authority lucide-react radix-ui react
```

Some components require additional peer dependencies, such as `@tanstack/react-table`, `@dnd-kit/*`, `react-hook-form`, `react-day-picker`, `react-aria-components`, `motion`, `next-themes`, `sonner`, `vaul`, `cmdk`, `input-otp`, `embla-carousel-react`, `@headless-tree/core`, and `react-resizable-panels`.

## Usage

```tsx
import { Button, Card, CardContent, CardHeader, CardTitle } from "@kumix/ui";
import "@kumix/ui/css";

export function Example() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Kumix UI</CardTitle>
      </CardHeader>
      <CardContent>
        <Button>Continue</Button>
      </CardContent>
    </Card>
  );
}
```

Hooks can be imported from the hooks entry:

```tsx
import { useMediaQuery, useViewport } from "@kumix/ui/hooks";
```

## Styling

Components use Tailwind utility classes and design-token CSS variables such as `--background`, `--foreground`, `--primary`, `--border`, and related semantic tokens.

The `@kumix/ui/css` entry currently emits the package CSS source directive from `src/style.css`. Consumers are expected to include the package source in their Tailwind build setup so component classes are generated by the consuming application.

## Repository Layout

- `packages/ui` - publishable Radix-based React component package (`@kumix/ui`).
- `packages/shadcn` - publishable Base UI + shadcn registry React component package (`@kumix/shadcn`).
- `packages/mcp` - private MCP helper package, ignored by Changesets.
- `apps` - application workspace placeholders.

## Development

Use Bun only. The repo is configured with `packageManager: bun@1.3.14` and requires Node `>=24`.

```bash
bun install
bun run build
bun run lint
bun run types:check
bun run test
bun run test:coverage
```

Run commands for `@kumix/ui` only:

```bash
bun run build --filter=@kumix/ui
bun run types:check --filter=@kumix/ui
bun run test --filter=@kumix/ui
bun run test:coverage --filter=@kumix/ui
```

The UI package build runs `tsdown`, emits declarations, emits `dist/style.css`, checks package compatibility with Are The Types Wrong, and validates publish output with `publint`.

## Testing

Tests use Vitest with the `jsdom` environment. Coverage reports are generated with V8 coverage:

```bash
bun run test:coverage --filter=@kumix/ui
```

Coverage thresholds are not enforced yet. The current tests are focused on baseline hooks behavior, selected accessibility safeguards, packaging-sensitive behavior, and primitive component smoke coverage.

## Publishing

This repository uses Changesets. The release script publishes non-private packages under `packages/**`.

```bash
bunx changeset
bun run version
bun run release
```

Publishing requires `NPM_TOKEN` in CI.
